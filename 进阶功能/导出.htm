输出

 

本章节介绍如何使用 Adobe RoboHelp 的输出进阶功能，包括输出响应式 HTML5、PDF 和移动应用。

输出响应式 HTML5

什么是响应式 HTML5

输出响应式 HTML5 使你的项目在各种屏幕尺寸的设备上使用。这意味着你可以发布经过优化的内容，以适合iPad，iPhone等流行设备的屏幕。

配置响应式 HTML5 输出预设

General

使用以下选项来指定基本输出设置：

  • Title 指定生成的输出的标题。可以在标题中使用变量，请输入 Ctrl + 1 并从下拉列表中选择一个变量，然后按 Enter

  • Favicon 指定一个图标与输出关联。此图标出现在输出的浏览器选项卡上。单击  浏览并选择所需的图标

  • Output Path 指定输出的位置。要选择一个位置，请单击 

  • Start Page 指定 RoboHelp 可以用来生成输出 URL 的页面名称和扩展名（.htm或.html）。例如，指定 index.html

  • Language 如果生成的输出的UI语言需要与项目设置中指定内容的语言不同，请使用下拉列表为输出预设指定语言

  • Encoding 使用下拉列表指定用于内容的字符编码格式的类型

  • Use lowercase file names 选择以生成所有具有小写文件名的输出文件

  • Post Generation Script 要在生成输出后运行自定义脚本，请从下拉列表中选择脚本 .js 文件

Content

使用以下选项来指定与内容相关的输出设置：

  • Table Of Contents 使用下拉列表选择要包含在输出中的目录。此下拉列表显示项目中可用的目录。默认情况下，此下拉列表中的第一个目录被选中

  • Browse Sequence 使用下拉列表查看项目中定义的浏览顺序。你可以选择多个浏览序列。然后，将所选的浏览序列合并为输出。如果与所选浏览顺序有冲突，则优先选择第一个选项

  • Default Topic 单击  以选择打开输出时显示在 Topic 面板中的主题。默认情况下，输出选择的 Table of Contents 中第一个主题是默认主题

  • Include Index 选择此设置可将索引页面包括在输出中

  • Glossary 使用下拉列表选择要包含在输出中的词汇表。词汇表可帮助你的用户使用相应的定义快速检索项目中的相关术语

  • Condition Expression 使用下拉列表为你的输出指定条件表达式。通过此设置，你可以根据所需的输出或用户群类型轻松地包含或排除内容。单击  以编辑选定的条件表达式。你也可以在下拉列表中选择 None 以不指定任何条件表达式

  • Dynamic Content Filter 使用下拉列表选择动态内容过滤器，以便最终用户可以过滤生成的输出中的内容。单击  以编辑选定的动态内容过滤器。或者，在下拉列表中选择 None 以不指定动态内容过滤器

  • Variable Set 使用下拉列表指定在此输出中使用的变量集

Layout

使用以下选项来配置输出的外观：

  • Master Page 使用下拉列表选择要生成的输出的母版页

  • Skin 使用下拉列表选择要在输出中应用的皮肤

  • Show Thumbnail for Images 选择此字段可显示输出图像的缩略图：

    • Create Thumbnail for images 选择此字段可从输出中的现有图像生成图像缩略图。这些图像可以在用户单击时放大

    • Style Existing Images as Thumbnails 选择此字段可将现有图像设置为输出中的缩略图的样式。这些图像可以在用户单击时放大

Search

使用以下选项为你的输出指定与搜索相关的设置：

  • Use topic description as search context 选择此设置可在搜索主题结果中显示带有标题的主题描述。在 Set a Character limit 中，输入要在主题描述中显示的最大字符数

  • Enable auto complete in search box 选择此选项可根据用户键入的初始字符显示搜索查询的预测选项

  • Auto correct search query 选择此选项可更正在搜索过程中键入的拼写错误的术语。此设置最多可在搜索过程中键入的每个单词中更正一个字符。这样即使您输入了拼写错误的字词，也可以确保您收到相关的搜索结果

  • Show definitions from glossary 选择此选项可显示为此输出选择的词汇表中搜索词的定义。如果搜索词与词汇表词匹配，则会显示该词的定义

  • Generate XML sitemap 选择此设置可为你的已发布输出生成站点地图。发布的输出中的 sitemap.xml 文件包含站点地图。您可以将此文件提交到搜索引擎，以改善你网站的搜索结果

  • Select file types to exclude from search 你可以从搜索中排除特定类型的文件，例如PDF，Word和Excel。搜索过程中将不包括所选文件中的内容

Publish

  • 你可以将输出发布到 RoboHelp Server，SharePoint Online、FTP、SFTP 或文件系统。为了能够发布你的输出，首先要为适当的服务器类型配置一个发布配置文件

  • 保存发布配置文件后，RoboHelp 在“发布”选项卡中显示该配置文件。要发布您的输出，请选择适当的发布配置文件，然后单击 

  • 默认情况下，自上次发布以来，仅使用选定的发布配置文件发布已修改的文件

  • 要发布所有文件，请选择 Republish All 并单击  。要查看发布日志，请在 Output Presets 面板中，针对适当的输出预设，选择  > View Publish Log

生成响应式 HTML5

  1. 在项目的创作窗口中，单击标准工具栏右上角的 Quick Generate 图标  。或者，在 Output 菜单中选择 Quick Generate 

  2. 在 Quick Generate 对话框中选择一个响应式 HTML5 输出预设

  3. 要配置设置，请执行以下操作之一：

    • 在 Quick Generate 对话框中，单击 Edit Settings 图标 

    • 点击标准工具栏左侧的 Output 选项卡

  4. 在输出工具栏中，单击 Output Presets。在 Output Presets 面板中，执行以下操作之一：

    • 双击所需的响应式HTML5输出预设

    • 单击所需输出预设旁边的图标  ，然后选择 Edit

  5. 查看配置响应式 HTML5 输出预设

  6. 要保存设置，请单击 Output 视图中标准工具栏左上角的 Save 图标 

  7. 单击 Output Presets 面板中的 Generate Preset 图标 

  8. 输出生成完成后，在 Output Presets 面板中单击  以查看输出

输出 PDF

什么是 PDF

输出 PDF 可帮助你将项目输出分发给数百万用户。Adobe PDF 或可移植文档格式是一种用于查看，共享和打印内容的通用文件格式。 PDF 保留字体，图像以及使用不同应用程序和平台创建的内容的布局。它们紧凑，易于使用链接导航，并且在打印工作流程中有效。

配置 PDF 输出预设

General

使用以下选项来指定基本输出设置：

  • Title 指定生成的输出的标题。可以在标题中使用变量，请输入 Ctrl + 1 并从下拉列表中选择一个变量，然后按 Enter

  • Output Path 指定输出的位置。要选择一个位置，请单击 

  • PDF File 指定要生成的输出文件的名称和扩展名（.pdf）。例如，指定 Printed_Documentation.pdf

  • Language 使用下拉列表为输出中的索引，词汇表和目录标题指定语言。如果要更改项目设置中指定的语言，请指定此设置

  • Post Generation Script 要在生成输出后运行自定义脚本，请从下拉列表中选择脚本 .js 文件

Content

  • Table Of Contents 使用下拉列表在项目中选择所需的目录。输出中仅显示所选目录中的主题

  • Include Table Of Contents 在 PDF 输出中包括目录页面。取消选择此字段可从输出中排除目录页面

  • Include All Referenced Topics 包括不属于目录的主题，但从目录中的主题引用的主题。选择此字段以包括PDF中的参考主题

  • Cover Page 单击  以选择查看输出时首先显示的文件

  • Include Index 选择此设置可将索引页面包括在输出中

  • Glossary 使用下拉列表选择要包含在输出中的词汇表。该词汇表可帮助你的用户使用相应的定义快速检索项目中的相关术语

  • Condition Expression 使用下拉列表为你的输出指定条件表达式。通过此设置，你可以根据所需的输出或用户群类型轻松地包含或排除内容。单击  以编辑选定的条件表达式。你也可以在下拉列表中选择 None 以不指定任何条件表达式

  • Variable Set 使用下拉列表指定在此输出中使用的变量集

  • Include Expanding Text content 选择此选项可在 PDF 输出中包括扩展文本；否则，仅扩展标题包括在输出

  • Include Drop-down Text content 选择此选项可在 PDF 输出中包括下拉文本；否则，只有下拉标题包括在输出

Layout

使用以下选项可以指定输出中外观的详细信息：

  • Master Page 使用下拉列表选择要生成的输出的母版页

  • Specify separate headers and footers 选择此选项可指定母版页，以为输出中的 Topics、TOC、Index 和 Glossary页面类型分配不同的页眉和页脚

PDF

使用以下选项指定页面大小，目录深度和页面边距设置：

  • Page Size 使用下拉列表为输出选择所需的页面大小。你可以从A4，A3，Letter等选项中进行选择，也可以选择自定义以指定自己的页面宽度（以毫米为单位）和页面高度（以毫米为单位）

  • Page Margin 以毫米为单位指定所需的页边距

  • Header Bottom Margin 指定页边距（以毫米为单位）以控制标题和输出中正文内容之间的间距

  • Footer Top Margin 指定页边距（以毫米为单位）以控制正文内容和输出中页脚之间的间距

Publish

  • 你可以将输出发布到 RoboHelp Server，SharePoint Online、FTP、SFTP 或文件系统。为了能够发布你的输出，首先要为适当的服务器类型配置一个发布配置文件

  • 保存发布配置文件后，RoboHelp 在“发布”选项卡中显示该配置文件。要发布您的输出，请选择适当的发布配置文件，然后单击

  • 默认情况下，自上次发布以来，仅使用选定的发布配置文件发布已修改的文件

  • 要发布所有文件，请选择 Republish All 并单击  。要查看发布日志，请在 Output Presets 面板中，针对适当的输出预设，选择  > View Publish Log

输出移动应用

什么是移动应用

在 RoboHelp 中，移动应用程序输出预设可让你生成 Android 或 iOS 移动应用程序作为项目输出。要使用 RoboHelp 生成移动应用程序，你需要一个 PhoneGap 帐户。如果你没有 PhoneGap 帐户，则可以在 RoboHelp 中的输出预设设置的 PhoneGap ID 部分中创建一个。

App Details

  • Package 指定移动应用程序的名称。常见的命名约定是反向使用公司域名，后跟应用程序的名称。例如 com.adobe.myapp。PhoneGap 使用它来唯一标识应用程序
    为避免 PhoneGap 中使用的第三方工具出现问题，请使用英文字母，下划线字符和句点作为分隔符

  • Version 应用程序的 PhoneGap 版本号

  • Version Code (Android only) 该应用的 Android 版本号。这是一个正整数，每次更新应用程序时都会增加一个值。当你将应用程序上载到 Android 商店（例如 Google Play）时，当你更新版本号时，系统会通知用户应用程序的更新

  • Title 移动应用程序的名称。当用户安装应用程序时，它会显示在Google Play商店或App Store中以及移动设备上

  • Description 指定应用程序的PhoneGap描述

  • Icon 指定与应用程序关联的图标。当用户安装应用程序时，它会显示在 Google Play 商店或 App Store 中以及移动设备上

  • Splash Screen 指定应用程序的初始屏幕

Phonegap ID

  • User ID 输入 PhoneGap 帐户的用户 ID

  • Password 输入 PhoneGap 帐户的密码

  • Create PhoneGap Account 如果没有 PhoneGap 帐户，请单击此按钮。然后，你将被定向到 PhoneGap 计划页面

  • Validate User ID 单击此按钮测试您的 PhoneGap 凭据。如果失败，请访问 phonegap.com 并使用相同的凭据登录。登录到 phonegap.com 之后，请重新测试 RoboHelp 中的凭据

Signing Key

  • Generate iOS App 选择此选项以生成一个 iOS 应用作为你的输出并指定以下内容：

    • Digital Certificate 点击  上载应用程序的数字证书

    • Certificate Password 输入数字证书的密码

    • Provisioning Profile 点击  指定数字证书。数字证书包含已注册测试设备的详细信息。如果 APP 未发布在 App Store 上，则只能安装在已注册的测试设备上

  • Generate Android App  选择此选项以生成一个 Anroid 应用作为你的输出并指定以下内容：

    • Keystore File 单击  以上传密钥库文件以验证应用程序。签署 Android 应用程序时，即表示使用凭证为应用程式添加标记。你可以使用同一密钥来标记多个应用程序

    • Keystore Password 指定密钥库文件的密码

    • Certificate Alias 指定密钥库的别名

    • Certificate Password  指定密钥库别名的密码
