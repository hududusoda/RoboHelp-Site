发布

 

本章节介绍如何使用 Adobe RoboHelp 的发布进阶功能，包括导入 FrameMaker 文件、发布到 Robohelp Server 和 FTP、SFTP或 File System 服务器。

导入 FrameMaker 文件至项目

导入 FrameMaker 文件

你可以将 FrameMaker books （.book）和 documents （.fm, .mif）导入至 Robohelp 项目中。导入文档中引用的所有资产都放置在资产文件夹的相应子文件夹中。例如，样式表放置在 css 子文件夹中，图像放置在 images 子文件夹中。

  1. 目录面板中选择 File > Import > FrameMaker Document

  2. 在弹出的选择框中输入 FrameMaker 文件地址并进行设置

  3. 点击 Import 即可导入 FrameMaker 文件

FrameMaker 导入设置

导入 FrameMaker 文件时可以进行相关设置。

General 

选择 general 选项卡为导入的内容进行基本设置。

  • FrameMaker Document 点击 选择需要导入的文件

  • CSS file for style mapping 点击  选择外部样式表，使 RoboHelp 样式映射 FrameMaker 样式

  • Start new topic from style 根据所选 FrameMaker 文档中的内容创建 topic

    • FrameMaker Style 从 FrameMaker 文档中选择 RoboHelp 可用于创建 topic 的段落样式。例如，要为每个一级标题创建一个 topic ，可选择一级标题

    • Topic Name Pattern 指定新主题的命名模式

      • <$ filename_no_ext>-<n> 创建不带扩展名的文件名并按顺序包含数字。例如，如果ABC.fm是导入的FrameMaker文档，则新主题的文件名将变为ABC1，ABC

      • <$ filename_no_ext>-<$ paratext> 创建不带扩展名的文件名，并包含第一段的初始文本

      • <$ paratext> 将应用所选样式元素的初始文本用于创建文件名。默认情况下设置此选项

    • Use only ASCII characters in the generated file names 选择此字段可将新主题文件名中的非ASCII字符转换为ASCII字符

  • Save Settings 保存 FrameMaker Import Settings 为设置文件（.fsf）

  • Load Settings 点击 选择需要导入的设置文件（.fsf）

Character

选择 character 选项卡为导入的内容指定字符样式。在此选项卡中，可以查看以下列：

  1. FrameMaker Style 在 General 选项卡的 FrameMaker Document 字段中选择的FrameMaker 文档中的样式

  2. 在 RoboHelp 中，从 General 选项卡的 CSS file for style mapping 字段中指定的 CSS文件填充的样式

对于每种 FrameMaker 样式，在 RoboHelp 中单击以选择其他样式并将其映射到相应的 FrameMaker 样式。 如果没有样式与相应的 FrameMaker 样式匹配，则RoboHelp 提供两个样式选项：

  1. Import style （默认）将样式从选定的 FrameMaker 文档导入到 RoboHelp

  2. Discard content 放弃应用所选样式的所有内容

可以分别在 FrameMaker Preview和 RoboHelp Preview部分中预览选定的 FrameMaker 样式和映射样式

Paragraph

选择 paragraph 选项卡为导入的内容指定段落样式。在此选项卡中，可以查看以下列：

  1. FrameMaker样式在 General 选项卡的 FrameMaker Document 字段中选择的FrameMaker文档中的样式

  2. 在 RoboHelp 中，从 General 选项卡的 CSS file for style mapping 字段中指定的 CSS 文件填充的样式

对于每种 FrameMaker 样式，在 RoboHelp 中单击 以选择其他样式并将其映射到相应的 FrameMaker 样式。 如果没有样式与相应的 FrameMaker 样式匹配，则RoboHelp 提供两个样式选项：

  1. Import style （默认）将样式从选定的 FrameMaker 文档导入到 RoboHelp

  2. Discard content 放弃应用所选样式的所有内容

可以分别在 FrameMaker Preview 和 RoboHelp Preview部分中预览选定的FrameMaker 样式和映射样式

Table

选择 table 选项卡为导入的内容指定表样式。在此选项卡中，可以查看以下列：

  1. FrameMaker 样式在 General 选项卡的 FrameMaker Document 字段中选择的 FrameMaker 文档中的样式

  2. 在 RoboHelp 中，从 General 选项卡的 CSS file for style mapping 字段中指定的 CSS 文件填充的样式

对于每种 FrameMaker 样式，在 RoboHelp 中单击 以选择其他样式并将其映射到相应的 FrameMaker 样式。 如果没有样式与相应的 FrameMaker 样式匹配，则RoboHelp 提供两个样式选项：

  1. Import style （默认）将样式从选定的 FrameMaker 文档导入到 RoboHelp

  2. Discard content 放弃应用所选样式的所有内容

可以分别在 FrameMaker Preview 和 RoboHelp Preview 部分中预览选定的FrameMaker 样式和映射样式

Cross Reference

选择 Cross Reference 选项卡为导入的内容指定交叉引用样式。在此选项卡中，可以查看以下列：

  1. FrameMaker Style 在 General 选项卡的 FrameMaker Document 字段中选择的 FrameMaker 文档中的样式

  2. 在 RoboHelp 中显示样式的当前定义。 默认情况下，此列是从导入的FrameMaker文档中的定义填充的。 您可以通过使用常量和变量的组合来修改定义。 要指定变量，请键入<并从下拉列表中选择一个变量，或键入变量名称

可以分别在 FrameMaker Preview 和 RoboHelp Preview 部分中预览选定的FrameMaker 样式和映射样式。

Images

选择 images 选项卡为导入的内容指定图像样式。在此选项卡中，使用以下字段：

  • Do Not Regenerate Images 果要重新导入 FrameMaker 文档并且不需要更新导入内容中的图像，请选择此字段

  • Import Reference Page Images 选择此字段可从导入的 FrameMaker 文档中的“参考页”导入图像

  • Preserve 3D Images 保留所选 FrameMaker 文档中的 3D 图像，即 3D 图像不会展平为 2D 图像。发布输出后，浏览器中将显示图像中的 3D 信息。默认情况下，此字段处于选中状态

  • Preferred Dimensions：

    • Width 和 Height 指定导入图像的首选宽度和高度。默认情况下，宽度和高度字段被链接为接受相同的值。切换以设置不同的宽度和高度值

    • Scale 指定缩放因素以缩放导入的图像

    • Maintain Aspect Ratio 选择此字段可以根据导入的 FrameMaker 文档中的图像保持纵横比

  • Margin 为图像周围的所有四个边距指定一个值。要为顶部、底部、左侧和右侧的每个边距指定不同的值，则切换  

    • Width 和 Height 指定导入图像的首选宽度和高度。默认情况下，宽度和高度字段被链接为接受相同的值。切换以设置不同的宽度和高度值

  • Maximum Dimensions
    由于“首选尺寸”可以通过 Scale 和 Maintain Aspect Ratio设置覆盖，因此请使用此部分设置导入文档中图像尺寸的最大限制

  • Borders 控制使用此部分应用于导入图像的标准 HTML 边框的属性

    • Border Style 从下拉列表中选择边框样式。 默认为无

    • Color 单击 以选择边框颜色。 默认为黑色。 或单击 以从屏幕上选择一种颜色

    • Width 指定边框宽度

    • 从以下选项中选择要应用的边框：

      • 全部（默认）

      •  右边

      •  底边

      •  左边

  • 高级设置

    • Default Format 选择要将所有导入的图像转换为的默认图像格式

    • Color Bit Depth 根据选择的默认图像格式更改。 您可以从受支持的选项中选择深度

    • JPG Quality 指定一个值以设置所有导入的 JPG 图像的质量。 此字段根据大小和质量控制 JPG 压缩，例如，默认值100表示最佳质量和最小压缩率

    • Grayscale 将所有导入的图像转换为灰度

Advanced

选择 Advanced 选项卡为导入的内容指定高级设置。在此选项卡中，可以查看以下列：

  • Context sensitive help marker 如果要从 FrameMaker 文档中导入上下文相关帮助标记，请选择此字段并指定要导入的上下文相关帮助制作者的类型

  • Split into topics using marker 如果要基于 FrameMaker 文档中的特定标记类型创建主题，请选择此字段并指定标记类型

  • Keep all topics in one folder  如果要导入 FrameMaker book ，可选择此字段以将 book 中所有主题保留在同一文件夹中

  • Add all topics to Table of Contents 中选择此字段可将从选定 FrameMaker 文档创建的所有主题添加到正在创建的目录文件中

  • Cleanup inline HTML styles 如果导入的 FrameMaker 文档具有内联样式，请选择此字段以放弃此类样式

  • Import Glossary files 将所选 FrameMaker 文档中的所有词汇表信息导入到新的词汇表文件中。默认情况下选中此选项

  • Import TOC files 导入从 FrameMaker 书中引用的所有目录文件，这些文件正被导入到新的目录文件中。默认情况下选中此选项

发布到 Robohelp Server

什么是 Robohelp Server

Adobe RoboHelp Server 10 是基于服务器的帮助解决方案，可为你的帮助和知识库提供实时的用户反馈。 在 RoboHelp 中输出项目，然后将其发布到 RoboHelp Server。 RoboHelp Server 收集并记录有关用户在搜索内容时提出的问题以及用户浏览 topics 方式的数据。 结果以易于查看的图形格式显示，以便快速解释。

如何发布到 Robohelp Server

要发布到 RoboHelp Server，首先要设置发布的配置文件，提供有关 RoboHelp Server 实例的信息。 RoboHelp 支持响应式 HTML5 输出类型，你可以编辑 HTML5 输出预设，选择要使用的配置文件， 然后将内容发布到RoboHelp Server。

配置 RoboHelp Server 发布

  1. 目录面板中选择 Edit > Publish Profiles

  2. 点击创建新的配置文件

  3. 在服务器类型字段中，选择服务器的类型为 RoboHelp Server。其默认值为 SharePoint Online。

  4. 对于 RoboHelp Server 服务器类型，请执行以下操作：

    • Name 显示默认配置文件名称。如果需要，请修改名称

    • Address 指定要将项目发布到的 Robohelp 服务器的地址
      地址格式： http：// <服务器名称>：<端口号> / <上下文名称> /服务器

      • server-name 设置 RoboHelp Server 的计算机的主机名或 IP 地址。如果它在同一台计算机上，则可以将本地主机指定为服务器名称

      • port-number（可选） 运行 RoboHelp Server 的端口。默认值为8080，如果你将默认端口用于HTTP或HTTPS协议，则无需提供端口号

      • context-name 要发布去的环境。默认值为robohelp

    • User ID 和 Password 验证身份

    • Areas 在服务器上选择要发布内容的区域。要刷新区域，请单击 

  5. 点击 save 保存配置文件

发布到 Robohelp Server

  1. 点击标准工具栏中的 Output 选项卡。 Output Presets 是 Output 工具栏中的默认选择

  2. 在 Output Presets 面板中，双击一个 output preset 以对其进行编辑

  3. 在 output preset 编辑器中，选择 Publish 选项卡以查看跨项目创建的服务器

  4. 选择要向其发布内容的配置文件（服务器）

  5. 选择 Republish ALL 以重新发布整个内容。如果未选择此字段，则 RoboHelp 仅发布自上次发布以来的更新内容，而不发布整个内容

  6. 要发布到选定的配置文件（服务器），单击output preset ，然后在标准工具栏中单击  

  7. 要查看已发布的内容，在已发布配置文件的结果列中单击

发布到 FTP、SFTP 或 File System 服务器

如何发布到 FTP、SFTP 或 File System 服务器

你可以在 RoboHelp 中生成项目的输出，然后使用文件传输协议（FTP），安全文件传输协议（SFTP）或文件系统将输出发布到外部服务器。为此，首先设置发布配置文件，以提供有关 FTP，SFTP 或文件系统实例的信息。编辑输出预设，选择要使用的发布配置文件，然后发布内容。

配置 FTP、SFTP 或 File System 服务器发布

  1. 目录面板中选择 Edit > Publish Profiles

  2. 点击创建新的配置文件

  3. 在服务器类型字段中，选择服务器的类型为 FTP、SFTP、或 File System。其默认值为 SharePoint Online。

  4. 如果需要，在 Name 字段中，修改默认配置文件名称

  5. 对于 FTP 服务器类型，请执行以下操作：

    • Address 指定服务器的域名或其IP地址

    • Anonymous User 选择此字段以匿名用户身份发布内容。匿名用户不需要登录凭据。如果你不想以匿名用户身份发布，请在 UserID 和 Password 字段中输入登录凭据

    • Server Directory 在服务器上指定要将内容发布到的路径。例如，为根文件夹指定/，为文件夹和子文件夹指定/ folder1 / folder2。如果指定的子文件夹不存在，则RoboHelp会创建它并在新的子文件夹中复制文件

    • Port 指定端口。默认值为21

  6. 对于SFTP服务器类型，请执行以下操作：

    • Address 指定服务器的域名或其IP地址

    • User ID 和 Password SFTP服务器仅支持用户名和密码登录。输入你的登录凭据

    • Server Directory 在服务器上指定要将内容发布到的路径。例如，为根文件夹指定/，为文件夹和子文件夹指定/ folder1 / folder2。如果指定的子文件夹不存在，则RoboHelp会创建它并在新的子文件夹中复制文件

    • Port 指定端口。默认值为22

  7. 对于 File System 服务器类型，在 Destination Path 字段中，指定要将内容发布到的网络路径。仅支持绝对路径。 RoboHelp会在你提供的路径上复制文件，而无需为输出创建任何子文件夹

  8. 点击 Save 保存配置
