使用 Conditional Content

 

本章节介绍如何使用 Adobe RoboHelp 的 Conditional Content 即条件内容的进阶功能，包括其定义、条件标记、配置条件内容的输出预设、和优化和管理条件内容。

什么是条件内容（Conditional Content）

Conditional Content 即条件内容，帮助组织有效地使用单一来源内容，以满足各种文档目的和受众需求。 在Adobe RoboHelp中，通过将条件标记应用于内容来实现条件内容。 条件标记能让用户生成内容多样性的输出。

此外，管理标记组可以帮助对多个相关条件标记进行分组，以便为焦点组提供相关内容。 你可以使用标记组，如访问群体，平台，产品，属性和其他自定义组。 例如，使用受众组向开发人员，管理员等提供目标内容。 使用platform组将操作系统相关信息定向给适当的用户，例如 Windows 和 Mac 的系统需求。

创建和应用 Condition Tags

要在项目中实现什么是条件内容，你需要添加条件标记并将这些标记应用到内容中。 应用标签后，你可以预览带标签的内容，以检查应用的标签是否按预期工作。

添加 Condition Tags

AdobeRoboHelp 提供了两个默认条件标记-联机和打印。你可以添加新标记，并使用Condition Tags 对话框中的 Group 字段对一堆相关标记进行分组。例如，可以将初学者，中级，专家，开发人员，管理员和最终用户标记分组到受众组下。 你还可以选择背景和文本颜色来区分标签以及应用标签的内容。

要添加新标记，请执行以下操作:

  1. 单击 Author 选项卡

  2. 在 Author 工具栏中，单击 Condition Tags 图标

  3. 在 Condition Tags 面板中，单击右上角的New Tag图标

  4. 在 Condition Tags 对话框中，执行以下操作:

    • 从 Group 下拉列表中选择要在其中创建标记的组。 例如，要标记特定于受众的内容，请选择受众组

注意:
如果你从早期版本升级到Adobe RoboHelp 2019，则所有以前的标记都被分组在 Condition Tags 面板中的报告其他报告组下。 这是因为以前版本的Adobe RoboHelp不包含Group字段。 Adobe建议你将标记组织在适当的组下。 为此，请将其他组中的标记拖放到 Condition Tags 面板中的相应组中。

在 Condition Tags 面板中，组（如，受众和其他）以及每个组下的标记（如 Developer 和 Adventure），如图所示:

  • 在 Name 字段中指定标记名称

  • 在标识区域中，选择背景和文本颜色
    背景和文本颜色通过突出显示应用标签的内容块来帮助区分多个条件标签

  • 单击 Done

    此时将打开 Project Setting 对话框

配置条件内容的输出预设

在项目中实现了什么是条件内容之后，需要添加或配置要在发布过程中使用的条件表达式和筛选器。 条件表达式和动态内容筛选器都使用条件标记对内容进行粒度控制。 然后，配置要从中发布的输出预设，以包括适当的条件表达式和动态内容筛选器。 接下来，为配置的输出预设生成输出。

配置条件表达式

条件表达式条件表达式控制在输出中包含或排除带标记的 Source 内容。 根据表达式中使用的条件标记，输出中包含相应的 Source 内容。 可以使用OR，AND和NOT运算符在表达式中定义各种标记组合，以自定义输出中内容的包含或排除。 条件表达式不会与 Source 内容一起发布，也不会出现在输出中。

  1. 单击标准报告工具栏左侧的 Output 选项卡

    此时将打开输出视图

  2. 在 Output 工具栏中，单击 Condition Expression 图标

  3. 在Condition Expressions面板中，单击Add图标

  4. 在 New Condition Expression 对话框中，指定条件表达式的名称。 单击 Done 完成报告 

    新的条件表达式将添加到 Condition Expression 面板中

  5. 在 Condition Expression 面板中，双击以打开表达式

    Condition Tags 面板和 Condition Expressions 面板将打开

  6. 从 Condition Tags 面板中，拖动一个标记并将其放到 Condition Expressions 面板中

    该标记将从 Condition Tags 面板中删除，并作为条件显示在 Condition Expressions 面板中

     

    你可以根据需要删除多个标记

    你可以通过单击表达式上的 Delete 图标来删除该表达式

  7. 针对 Condition Expressions 面板中的每个条件，执行下列操作之一:

    • 选择Include在表达式中添加标记。 此操作包括输出中与标记关联的所有 Source 内容

    • 选择Exclude从表达式中排除标记。 此操作从输出中排除所有与标记关联的 Source 内容

  8. 若要组合多个条件，请使用AND和OR运算符

    若要确保表达式仅在满足所有条件时运行，请在条件之间使用AND运算符。 如果满足一个或多个条件，使用OR运算符让表达式运行

    下面的表达式显示加拿大区域设置的开发人员或最终用户内容

  9. 要保存更改，请单击标准报告工具栏中的 Save 

配置动态内容筛选器

动态内容筛选器动态内容筛选器控制输出中内容的显示。添加动态内容筛选器，然后通过添加条件标记对其进行自定义。筛选器将与标记的 Source 内容一起发布。 从输出中显示的已发布筛选器中，最终用户可以选择特定的筛选器来查看选择性内容并隐藏其余内容。 动态内容过滤器便于快速检索和消费特定信息，而不必遍历整个发布的内容。

  1. 单击标准报告工具栏左侧的 Output 选项卡，此时将打开输出视图

  2. 在 Output 工具栏中，单击 Dynamic Content Filters 图标

  3. 在 Filters 面板中，单击 Add 图标  

  4. 在 New Filterdialog 框中，指定动态内容筛选器的名称。 单击 Done 完成报告 
    新的动态内容筛选器将添加到报告筛选器报告面板中。 Condition Tags 面板和 Condition Tags 面板将打开。 筛选器编辑器显示新的动态内容筛选器

  5. 从 Condition Tags 面板中，拖动一个标记并将其放到 Condition Tags 面板中
    该标记将从 Condition Tags 面板中删除，并与其原始组一起出现在 Condition Tags 面板中。例如，报告US报告标记及其原始组报告 Others 报告出现在 Condition Tags 面板中

    如果不希望原始组与标记一起出现，请通过单击 Condition Tags 面板创建新的筛选器组。
    将显示一个具有可编辑标题报告新建组报告的新筛选器组。 根据需要重命名标题，例如，将标题重命名为US-ONLY
    从 Condition Tags 面板中，拖动一个标记，并将其放在 Condition Tags 面板中的新筛选器组下

  6. 若要为用户提供跨已发布输出中的筛选器组选择多个筛选器的功能，请选择 Allow Mutilple Selection 字段。 默认情况下取消选择此字段，以提供仅从每个筛选器组中选择一个筛选器的能力

  7. 若要保存更改，请单击标准报告工具栏中的 Save 

配置输出预设以生成条件内容

要能够生成条件内容，必须配置适当的输出预设，以使用特定的条件表达式和/或动态内容筛选器。

  1. 单击标准报告工具栏左侧的 Output 选项卡，此时将打开输出视图

  2. 在 Output 工具栏中，单击 Output Presets 图标

  3. 在 Output Presets 面板中，双击一个输出预置以打开它。Output Preset Editor面板打开，默认选项为 General 

  4. 在 Output Preset Editor面板中，单击 Content 

  5. 在 Content Editor面板中，执行以下操作:

    • 从 Condition Expression 下拉列表中，选择一个条件表达式。 默认情况下，选择 none

    • 从Dynamic Content Filters 下拉列表中，选择一个动态内容筛选器。 默认情况下，选择 none

  6. （可选）你可以通过单击相应的编辑来编辑选定的条件表达式和动态内容筛选器

    图标反对它

  7. 若要保存更改，请单击标准报告工具栏中的 Save 

使用配置的输出预设生成输出

  1. 单击标准报告工具栏左侧的 Output 选项卡

    此时将打开输出视图

  2. 在 Output 工具栏中，单击 Output Presets 图标

  3. 在 Output Presets面板中，选择要发布的已配置输出预置，然后单击Generate preset图标
    或者，根据你希望使用的已配置输出预设，选择 Options > Generate
    在生成输出时，会根据输出预设显示一个进度条
    创建新项目时，默认情况下，输出文件保存在用户区域内的输出文件夹中:
    <user_area>/Documents/My RoboHelp Projects/outputs/<Folder_Name>/<Output_Preset_Name>
    例如:
    C:/users/jdoe/documents/my RoboHelp projects/outputs/travel the wirls/responsive HTML5

  4. 要保存更改，请单击标准报告工具栏中的 Save 
    更改输出文件夹路径的选项可用于所有输出预设，移动应用程序除外

  5. 要查看输出，请在 Output Presets 面板中，对照发布时使用的输出预置，选择 Option > View 。根据输出类型，它将在其默认查看器中打开

  6. （可选）要查看输出日志，请在 Output Presets 面板中，对照发布时使用的输出预置，选择 Option > View Log 

优化和管理条件内容

学习如何通过管理条件标签，条件表达式和动态内容筛选器来优化什么是条件内容。 查看生成条件报告的步骤。

管理条件标记

你可以添加，编辑，删除和更改条件标记的组。

  1. 在 Author 工具栏中，单击 Condition Tags 图标

  2. 若要添加标记，请参见添加条件标记

  3. 编辑标记:

    • 在 Condition Tags 面板中，双击要编辑的条件标记。 或者，右键单击条件标记，然后选择 Edit

    • 在报告编辑条件标记报告对话框中，根据需要修改标记组，标记名称以及背景和文本颜色

    • 单击 Done 完成报告。保存对条件标记所做的更改

  4. 删除标记:

    • 在 Condition Tags 面板中，右键单击条件标记，然后单击 Delete 

    • 在 Confirm 对话框中，单击 Yes 确认删除。 条件标记将被删除

  5. 更改标记的组:
    在 Condition Tags 面板中，标记（如Developer和Adventure）及其各自的组（如，受众和其他人）如图所示:

    若要更改条件标记的组，请将其从当前组中拖动，然后将其放到目标组下。 在目标组中，新添加的标签按字母顺序列出

6．要保存更改，请单击标准报告工具栏中的 Save 

管理条件表达式和动态内容筛选器

管理条件表达式

  1. 单击标准报告工具栏左侧的 Output 选项卡
    此时将打开输出视图

  2. 在 Output 工具栏中，单击 Condition Expression 图标

  3. 若要添加/配置表达式，请参见配置条件表达式和动态内容筛选器

  4. 编辑表达式:
    在 Condition Expressions 面板中，按要求执行以下操作:

    • 添加或删除条件标记

    • 修改运算符-include/exclude and and/or

    • 要重命名表达式，请在 Condition Expressions 面板中选择表达式，然后选择 Options > rename

    • 要删除表达式，请在 Condition Expressions 面板中选择表达式，然后选择 Options > delete

    • 若要保存更改，请单击标准报告工具栏中的 Save 

管理动态内容筛选器

  1. 单击标准报告工具栏左侧的 Output 选项卡
    此时将打开输出视图

  2. 在 Output 工具栏中，单击报告筛选器报告图标

  3. 若要添加/配置动态内容筛选器，请参见配置条件表达式和动态内容筛选器

  4. 编辑筛选器:
    在 Condition Tags 面板中，按要求执行以下操作:

    • 修改筛选器组

    • 添加或删除条件标记

    • 要删除筛选器，请在报告筛选器报告面板中选择该筛选器，然后选择 Option > Delete 

    • 若要保存更改，请单击标准报告工具栏中的 Save
