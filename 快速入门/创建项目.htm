创建项目

 

项目是 Adobe RoboHelp 的核心，topic 是项目的基本元素。用 Robohelp 创作内容包括：向项目中添加 topic 、添加全局导航、配置项目属性、生成所需文件。RoboHelp 的经典输出是一种或多种格式的帮助文档，例如响应 HTML5 和 PDF。RoboHelp 项目文件的扩展名为.rhpj。

除项目文件外，RoboHelp 还有各种项目组件，例如 topic、table of contents、variables、skin 和 master page。用户无需直接操作项目文件或组件文件，在 RoboHelp 中的任何编辑行为都会自动更新到相应文件。

了解项目组件

        下表列出了 RoboHelp 项目的各个组件：

组件名 描述

Topic

RoboHelp 中内容结构的基本元素。topic 可以包含文本、多媒体（如图像和视频）、链接等。设计 topic 主题时请确保其易于重用，避免 topic 太长或太短。

文件夹和子文件夹

用于组织项目内容。创建文件夹和子文件夹来组织 topic 和 assets，例如图像，视频和样式表。

图像和多媒体

可以在顶部工具栏选择向 topic 中插入图像和多媒体，以丰富内容。

 

Table of contents (TOC)

帮助呈现项目内容的层次结构，创作过程中可以浏览 TOC 并选择文件夹或主题进行查看和编辑。在发布的文档中，最终用户可以使用 TOC 快速跳转、查看所选 topic 的内容。

索引 (Index)

允许最终用户使用指定的关键字或短语在帮助系统中导航。这些关键字或短语需要与特定主题相关联。索引可以包含多个级别的关键字以及对其他关键字的交叉引用。

Glossary

显示文档中的术语列表。Robohelp 支持设置条件筛选术语、链接到项目外部的词汇表文件、在最后生成的文档中包含多个词汇表文件。

Condition tags

定义内容的子集，并在同一项目中生成多个文档，便于针对不同需求（例如不同的目标受众和交付格式）管理内容。

 

Variables

在此存储用于项目多处的内容，此处一更新，所有包含相同内容的项目组件均会改变，有助于快速进行全局更新并确保一致性。一个很好的用例是存储诸如产品名称和发布日期之类的信息。

Snippets

创建和管理复用内容。用 Global snippets 识别指定内容，仅需一步即可更改所有 topic 的相应部分，从而节省时间并促进有效的内容更改。

 

Master pages

提供用于 topic 布局的模板，并帮助控制内容样式。在 topic 创建过程中可将master page 应用于所有 topic 或正在创建编辑中的 topic，然后在预览模式下查看效果。

 

Browse sequences

文件的线性列表，可用于从逻辑上导航到下一个相关 topic。支持的操作有添加、编辑、删除。在 browse sequences 中，可以在“Contents”面板和 TOC 中拖动 topic主题以更改顺序。

报告 (Report)

生成各种报告以查看、分享没有连上的链接、未使用的文件、缺少的 topic 等。

导出预设

设置输出格式和内容布局，还可以设置输出文档的搜索引擎配置，例如在搜索框中启用自动更正，显示词汇表中的定义等。

Skins

设置 HTML 输出的外观。

标记语言 (Labels)

可以编辑和删除语言标记，也可以撤消重做，还可以将语言标记导出为 XML 进行本地化，并导入特定语言的翻译副本。

 

Variable sets

通过覆盖 variable 的值来自定义 variables 用于特定输出。在新创建的variable set 中，列出了项目中的所有 variable。在此列表中，可以覆盖原值为特定输出定制的 variable。可以创建不同的 variable set，以针对不同的特定输出。

 

条件表达式

(Condition expressions)

用于控制和标记输出文档中要包含或排除的项目内容。根据表达式中使用的条件标签，相应的源内容将包含在最终输出的文档中。可以用 OR、AND 和NOT 运算符在表达式中连接各种标记组合，以自定义输出内容。条件表达式不会与源内容一起发布，也不会出现在最终的输出文档中。

Dynamic content filters

控制输出文档中内容的显示。最终用户可以选择特定的 filter 来查看匹配内容并隐藏不相关内容，有助于快速检索和使用特定信息，而不用遍历整个文档。

 

停用词 (Stop words)

向停用词列表添加常用又无实义的词，例如“ a”或“ the”。在搜索过程中，RoboHelp 会忽略停用词表中的词，以显示最佳且相关的搜索结果。RoboHelp 支持添加、编辑和删除不同语种的停用词，且为每种语言都创建了一个停用词列表文件。

 

同义词 (Synonyms)

可以为一个词定义多个同义词，应对搜索时的多词同义。例如，将“光标”和“鼠标”定义为同义词，检索“鼠标”时，RoboHelp 将返回所有包含“光标”的 topic，并在文内高亮显示该词。RoboHelp 支持添加、编辑和删除不同语种的同义词，且为每种语言都创建了一个同义词列表文件。

样式表 (Style sheets)

存储和定义用于控制内容外观的样式。为内容的不同元素（例如，不同的标题级别）定义样式，然后在项目或 topic 级别应用样式表。

新建项目

  1. 单击 File > New Project 或用键入 Alt+Ctrl+N，弹出新建项目的对话框

  2. 在对话框中设置项目名称、语言和存储地址

  3. 单击 Create，完成创建

保存项目

  • 单击 File > Save All 或键入 Ctrl+S，保存所有项目文件更改

  • 单击工具栏左上方的 ，保存所有项目文件更改

  • 单击 File > Save 或键入 Alt+Ctrl+S，保存当前 topic 更改

设置项目配置

项目创建后，可以更改项目名称、语言，设置 topic 状态、颜色、标签分组和版本控制。

  1. 启动 RoboHelp 并打开项目，单击 File > Project settings 或键入 Ctrl+Shift+/

  2. 在项目配置对话框中的 General、Topic Status、Colors 等选项卡中进行设置

    选项卡 操作

     

     

     

    General

    • 在 Project Title 一栏中编辑项目名称

    • 在 Language 一栏中点击 选择项目语言

    • 在 Default Stylesheet 一栏中点击  选择项目所需的样式表

    • 在 Default Word/FrameMaker Import Settings 一栏中选择导入 Word或FrameMaker 文件时的配置信息

    • 勾选 Do not allow inline formatting 以避免使用新的内联样式，保持之前的内联样式不变

    Topic Status

    选择 topic 状态，例如 Draft、In Progress 和 Ready for Review 来提示 topic 的创作阶段，支持自定义、重命名、删除、排序状态

    Colors

    设置项目内容需要的颜色，支持添加、删除、重命名现有颜色列表的对象

    Tag Groups

    对 Condition tags 进行分组管理，支持添加、删除、重命名现有分组，但如果标签组包含了Condition tags，则无法删除

    Version Control

    查看项目文件路径和版本控制、协作方式 

  3.  输入所需的设置后，点击 Done
